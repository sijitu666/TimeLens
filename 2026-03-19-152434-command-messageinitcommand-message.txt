  2. SFT 训练阶段

  关键调整参数

  在 train_scripts/run_sft_qwen3_8b.sh 中：

  # ============ 原始配置（H20 8卡） ============
  global_batch_size=128
  batch_per_device=1
  num_devices=8
  deepspeed_config="scripts/zero3.json"
  total_tokens=14336
  fps_max_frames=448  # 自动计算: 14336/64*2

  # ============ L40建议配置 ============
  # 方案A: 使用L40 4卡，保持global_batch_size但调整其他参数
  global_batch_size=64      # 减半
  batch_per_device=1
  num_devices=4             # 4张L40
  deepspeed_config="scripts/zero3.json"
  total_tokens=7168         # 减半
  fps_max_frames=224        # 自动计算: 7168/64*2

  # 方案B: 单卡L40训练（调试用途）
  global_batch_size=8       # 很小
  batch_per_device=1
  num_devices=1
  deepspeed_config="scripts/zero1.json"  # 使用ZeRO-1节省显存
  total_tokens=3584         # 进一步降低
  fps_max_frames=112
  gradient_accumulation_steps=8  # 增加梯度累积

  DeepSpeed配置调整

  对于L40，可能需要调整 scripts/zero3.json：

  {
    "bf16": {
      "enabled": true
    },
    "zero_optimization": {
      "stage": 3,
      "offload_optimizer": {
        "device": "cpu",    // 对于L40，开启optimizer offload
        "pin_memory": true
      },
      "offload_param": {
        "device": "cpu",    // 对于L40，开启param offload  
        "pin_memory": true
      },
      "overlap_comm": true,
      "contiguous_gradients": true,
      "reduce_bucket_size": 5e8,
      "stage3_prefetch_bucket_size": 5e8,
      "stage3_param_persistence_threshold": 1e6
    },
    "train_batch_size": "auto",
    "train_micro_batch_size_per_gpu": "auto",
    "gradient_accumulation_steps": "auto"
  }

  运行命令

  # 4卡L40训练示例
  CUDA_VISIBLE_DEVICES=0,1,2,3 \
  global_batch_size=64 \
  num_devices=4 \
  total_tokens=7168 \
  bash train_scripts/run_sft_qwen3_8b.sh --model_path
  "/path/to/Qwen3-VL-8B-Instruct"

  3. GRPO 训练阶段

  GRPO训练比SFT更耗费显存，因为需要维护多个generations。

  # ============ 原始配置（H20 8卡） ============
  global_batch_size=64
  num_generations=8
  total_tokens=14336

  # ============ L40建议配置 ============
  # 4卡L40
  global_batch_size=32      # 减半
  num_generations=4         # 减半（对显存影响大）
  num_devices=4
  total_tokens=7168

❯ 视频时间定位 (VTG) 模型进阶优化科研计划                                       
                                                                                
核心目标                                                                        
                                                                                
在 TimeLens 提出的“交错文本时间戳 + 无思考 RLVR”基线方案基础上                  
，针对高帧率细粒度动作（如机器人操作误差的视频定位分析）与长视频 MLOps          
部署的痛点，对其时间编码机制与强化学习训练管道进行底层架构与策略的双重升级。    
                                                                                
Phase 1: 架构与时间编码重构 (分配至：架构与算法 Agent)                          
                                                                                
本阶段旨在解决离散文本时间戳带来的连续性丧失以及长视频带来的 KV Cache           
爆炸问题，确保模型在受限算力环境（如常规 Colab 实例）中依然能高效训练与推理。   
                                                                                
Task 1.1: 连续时间嵌入 (Continuous Temporal Embeddings) 的开发与集成            
                                                                                
行动项: 移除原有将时间戳（如 "10.2s"）强制转为文本 Token 的预处理逻辑。         
                                                                                
技术路线: 引入可学习的连续时间嵌入层，或针对时间轴设计一维旋转位置编码（1D-RoPE 
）。确保模型在处理高频、细微动作视频时，能捕捉到微秒级的数学连续递进关系。      
                                                                                
Task 1.2: 轻量级时序适配器 (Temporal Adapter) 研发                              
                                                                                
行动项:                                                                         
解决交错前缀带来的上下文冗余问题，为后续的流式处理与高并发推理打好基础。        
                                                                                
技术路线: 设计旁路的时序适配器（或轻量级状态空间模型如 Mamba                    
变体）来隐式传递帧间的时间状态。要求在消除显式时间 Token                        
的同时，保持特征层面的时间感知能力。设计时需同步评估该模块在未来转换为 ONNX 或  
TensorRT 格式时的算子兼容性。                                                   
                                                                                
Phase 2: RLVR 训练管道升级 (分配至：强化学习与数据 Agent)                       
                                                                                
本阶段旨在将原有单一、静态的强化学习策略升级为多维、动态的闭环反馈系统，提升模  
型在复杂长尾数据上的泛化表现 。                                                 
                                                                                
Task 2.1: 多维组合奖励函数 (Multi-dimensional Reward) 实现                      
                                                                                
                                                                                
行动项: 弃用单一的纯时间 IoU 奖励 $r(y) = IoU(\hat{S}, S^*)$                    
，避免语义正确但边界微小偏移的预测被过度惩罚。                                  
                                                                                
技术路线: 引入视觉-文本对齐模型（如 CLIP 或 SigLIP）。构建复合奖励函数：        
                                                                                
$r_{total} = \alpha \cdot r_{iou} + \beta \cdot r_{semantic}$                   
                                                                                
其中 $r_{semantic}$ 为预测时间段内的视频特征与查询文本的余弦相似度。设定        
$\alpha$ 和 $\beta$ 的动态退火策略。                                            
                                                                                
Task 2.2: 在线动态难度评估 (Online Dynamic Difficulty Estimation)               
                                                                                
                                                                                
行动项: 消除模型在 RL 训练前必须完整跑一遍离线推理（Offline                     
Inference）的工程冗余 。                                                        
                                                                                
技术路线: 建立在线评估机制。利用强化学习过程中的历史滑动平均奖励（EMA of        
Rewards）来动态更新每个样本的难度权重，实现在同一个 Training Loop               
中一边更新策略一边调整采样概率。                                                
                                                                                
Task 2.3: 分层难度感知采样 (Stratified Difficulty Sampling) 逻辑编写            
                                                                                
                                                                                
行动项: 扩充难度维度的衡量标准，不再局限于单一的 $1 - IoU$ 。                   
                                                                                
                                                                                
技术路线: 在高斯采样逻辑中                                                      
，融合多维因子：空间复杂度（通过目标检测置信度评估）、时序动态复杂度（通过光流  
变化率评估）以及语义复杂度。构建多因子加权的难度评估矩阵。                      
                                                                                
Phase 3: 评估与对齐 (分配至：测试与验证 Agent)                                  
                                                                                
Task 3.1: 基线对比与消融实验                                                    
                                                                                
使用 TimeLens-Bench 作为验证集                                                  
。严格按照单一变量原则，分别量化“连续时间嵌入”与“多维奖励函数”对最终 R1@0.5 和  
mIoU 指标的贡献 。                                                              
                                                                                
Task 3.2: 性能与显存 Profiling                                                  
                                                                                
监控引入时序适配器后的显存占用变化，记录并对比采用在线动态难度评估前后的总体训  
练时长耗损。这是一个科研计划方案,请你帮忙进一步细化,拆分成多个agent可以相互合作
